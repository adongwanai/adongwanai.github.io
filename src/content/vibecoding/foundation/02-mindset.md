# 第 2 章：心法 —— 从打工人到指挥官

> **章节导语**
>
> 你即将学到的，不是"如何给AI写更好的提示词"——那是第三章的事。
>
> 这一章要解决的是一个更根本的问题：**在你开口告诉AI"帮我做xxx"之前，你应该想清楚什么？**
>
> 90%的Vibe Coding失败，不是因为AI不够聪明，而是因为你自己都没想清楚要做什么。
>
> 读完这一章，你会获得一套产品经理级别的思维工具。从此以后，无论你想做什么——一个小工具、一个数据分析、一个自动化脚本，还是一个产品原型——你都会知道如何思考。

> **预计阅读时间**：完整阅读约 2 小时 | 快速通关约 30 分钟

## 2.1 核心心法：工作流设计 > Prompt

> **来自 @supermao 的金句**：
> "你的核心筹码是**工作流设计能力**，不是 prompt 本身。"

### 过去 vs 现在

| 维度 | 过去 | 现在 |
|------|------|------|
| **重点** | 优化每一句 Prompt | 设计完整工作流 |
| **思维** | 点对点 | 端到端 |
| **能力** | 会说话 | 会规划 |

### 什么是工作流？

**简单理解**：你要让 AI 完成一个任务，需要哪些步骤？

**举例**：做一个"每日新闻摘要"工具

| 步骤 | AI 做什么 |
|------|-----------|
| 1 | 抓取新闻网站 |
| 2 | 提取关键信息 |
| 3 | 生成摘要 |
| 4 | 推送到邮箱 |

**工作流设计** = 规划这4步，以及每一步用什么模型、什么工具。

## 2.2 一贵一贱：AI 资产配置

> **@supermao 核心观点**："贵的模型干高价值思考，便宜的模型干机械劳动。"

### 模型分工表

| 模型 | 定位 | 做什么 | 预算 |
|------|------|--------|------|
| **Claude Opus 4.5** | 贵 | 架构设计、复杂决策、Debug | 中高 |
| **GPT-5.1-Thinking** | 贵 | 推理分析、长文本 | 中高 |
| **Gemini 3 Pro** | 贱 | 日常生成、抓数据 | 免费/低 |
| **开源模型** | 贱 | 机械劳动、批量处理 | 0 元 |

### 实际应用

```
项目：做一个博客自动生成器

工作流设计：
1. [便宜] 用 Gemini 3 Pro 抓取目标网站内容
2. [便宜] 用开源模型做初步清洗
3. [贵] 用 Claude Opus 4.5 生成文章结构和内容
4. [便宜] 用 Gemini 3 Pro 做最后润色
```

## 2.3 七个思维模型

### 模型 1：JTBD (Jobs to be Done)

> **核心**：不是做"什么功能"，而是满足"什么需求"。

| 错误 | 正确 |
|------|------|
| "我要做一个待办清单" | "我想不再忘记重要的事" |
| "我要做一个计算器" | "我想快速算出每月开销" |

**练习**：把你想要的功能，翻译成你想要解决的问题。

---

### 模型 2：逆向思维

> **核心**：先想"最坏的结果"，再想"如何避免"。

**问题清单**：
- 这个想法最大的风险是什么？
- 用户最可能抱怨什么？
- 什么情况下会失败？

**举例**：做一个天气预报小程序

| 问题 | 解决方案 |
|------|---------|
| API 挂了怎么办？ | 加备用 API |
| 数据不准怎么办？ | 显示数据来源 |
| 没人用怎么办？ | 先给朋友用 |

---

### 模型 3：减法思维

> **核心**：MVP（最小可行产品），先做最核心的一个功能。

**原则**：
- 能删则删
- 能减则减
- 一个功能做到 100 分 > 十个功能做到 60 分

**举例**：做一个读书笔记 App

| MVP 版本 | 完整版本 |
|---------|---------|
| 记录一句话笔记 | 记录、标签、分享、导出 |
| 1 个功能 | 10 个功能 |

---

### 模型 4：故事思维

> **核心**：把你的想法变成一个故事。

**模板**：

```
有一天，[用户] 发现了一个问题：[问题描述]
他试了 [现有方案]，但 [它们的缺点]
于是他找到了 [你的产品]
[产品名] 帮助他 [解决了什么问题]
现在他 [变得怎么样了]
```

---

### 模型 5：灵魂三问

> **核心**：每个想法都要回答三个问题

| 问题 | 说明 | 例子 |
|------|------|------|
| **给谁用？** | 目标用户是谁？ | 考研学生 |
| **解决什么？** | 痛点是什么？ | 记不住单词 |
| **为什么用？** | 为什么不买别的？ | 更便宜/更快 |

---

### 模型 6：场景应用

> **核心**：想清楚用户会在什么场景下使用

**模板**：

```
用户画像：小明，25岁，考研学生
使用场景：
- 早上地铁上：背单词
- 午休时：复习错题
- 睡前：听发音

需求优先级：
1. 单词记忆（高频）
2. 发音朗读（辅助）
3. 错题本（低频）
```

---

### 模型 7：问题发现

> **核心**：不仅要解决问题，还要发现更多问题

**方法**：
- 用户说"想要A" → 问"为什么想要A？"
- 用户说"太慢了" → 问"哪个环节慢？"

---

## 2.4 开发前检查清单

> **每次动手之前，花 5 分钟检查这 6 项**

| # | 检查项 | 问题 |
|---|--------|------|
| 1 | **给谁用？** | 目标用户是谁？ |
| 2 | **解决什么？** | 核心痛点是什么？ |
| 3 | **为什么用？** | 为什么不选其他方案？ |
| 4 | **最小功能？** | 最简版本做什么？ |
| 5 | **怎么验证？** | 怎么知道做成功了？ |
| 6 | **失败了会怎样？** | 最大风险是什么？ |

## 2.5 黄金心法 Prompt

> **直接复制使用**

```
你是我的超级个体 AI 指挥官。我想[描述你的目标]。

请帮我：
1. 拆解成 MRD（市场需求文档）和 PRD（产品需求文档）
2. 设计一条 Agentic 工作流（包含哪些模型、工具、迭代循环）
3. 给出"一贵一贱"模型配置建议
```

## 本章小结

| 核心概念 | 关键点 |
|---------|--------|
| **工作流设计** | 核心能力是规划，不是写 Prompt |
| **一贵一贱** | 贵模型做决策，便宜模型做执行 |
| **JTBD** | 关注需求，不是功能 |
| **MVP** | 先做一个功能做到 100 分 |
| **灵魂三问** | 给谁用？解决什么？为什么用？ |
